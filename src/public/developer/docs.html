Build with GoodOS.

Use the GoodOS Public API, JavaScript SDK, TypeScript definitions, OpenAPI spec, and Postman collection to connect GoodFleet, GoodQR, GoodAds, GoodBoost, GoodTrusts, and future apps to the same enterprise backend.

OpenAPI JSON API Reference JavaScript SDK TypeScript Types Postman Collection Console

API Key Auth

Use X-GoodOS-API-Key or Authorization: Bearer. Keys are scoped by permissions and app restrictions.

Scoped Access

Supported scopes include read:health, read:apps, read:storage, and execute:functions.

Project Ready

GoodOS is organized by organization, project, and environment for enterprise-style backend isolation.

Install the SDK

<script src="https://backend.goodos.app/sdk/goodos.js"></script>

JavaScript Example

const goodos = GoodOS.createClient({
  apiKey: "gak_your_key_here"
});

const health = await goodos.health();
const apps = await goodos.apps();

const result = await goodos.callFunction("health-check", {
  source: "developer-docs",
  message: "Hello from GoodOS"
});

Node / Bundler Example

const { createClient } = require("./goodos.js");

const goodos = createClient({
  apiKey: process.env.GOODOS_API_KEY
});

const response = await goodos.apps();
console.log(response);

Public API Endpoints

Method Endpoint Scope Purpose
GET/api/v1/healthread:healthCheck public API health and key context.
GET/api/v1/appsread:appsList apps allowed by the API key.
GET/api/v1/storage/bucketsread:storageList storage buckets.
GET/api/v1/storage/filesread:storageList storage files.
GET/POST/api/v1/functions/:slugexecute:functionsCall active HTTP Edge Functions.
Keep raw API keys private. Use separate keys for development, staging, production, and each app when possible.
