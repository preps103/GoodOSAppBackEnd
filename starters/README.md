# Goodbase starter applications

These starters use the canonical `GOODBASE_URL`, public project key, generated types, explicit tenant scope, and local-stack selection. Copy only the platform you need and follow its README. Mobile starters require attestation before push-token registration.

- `react/` — authenticated web client and offline sync
- `nextjs/` — server-rendered session boundary
- `flutter/` — encrypted mobile session/offline queue contract
- `swift/` — App Attest and APNs integration contract
- `kotlin/` — Play Integrity and FCM integration contract
- `multi-tenant/` — RLS policy and negative-test starter
